RequestItem RequestItem allows one to schedule a request to be sent to the phone. The request itself is built according to a description entered into the QXDM database or a user database. RequestItem waits for a response and if received, parses it according to the default description in the QXDM database or a user database (if such a description exists).
Syntax RequestItem "RequestName" Fields
"RequestName" is the name of the DIAG entity to request (entity must be either a DIAG request or subsystem dispatch request)
Fields is a list of space-separated field values used to build the request. This argument can be omitted if the request does not require field values.
Example RequestItem "Version Number Request"
Builds and sends a "Version Number Request" to the target and waits for and parses the response using the default database description

RequestItem "Extensible Parameter Retrieval Request" 1 1 100
Builds and sends a "Extensible Parameter Retrieval Request" with field values 1, 1, and 100 to the target and waits for and parses the response using the default database description
RequestItemFile RequestItemFile allows one to schedule a request to be sent to the phone using a file that contains input field values. The request itself is built according to a description entered into the QXDM database or a user database. RequestItemFile waits for a response and if received, parses it according to the default description in the QXDM database or a user database (if such a description exists).
Syntax RequestItemFile "RequestName" "FileName"
"RequestName" is the name of the DIAG entity to request (entity must be either a DIAG request or subsystem dispatch request)
"FileName" is the name of a file containing field values used to build the request. This argument can be omitted if the request does not require field values. The file should contain one line for each input field of field, value pairs, separated by the caret (^) character as in the following example:
    Number Of Digits^10
    Digits^2144951009
    Service Option^17
Note: QXDM searches for "FileName" based on the following search criteria:
1. Search using the file as-is
2. Search using the QXDM bin folder (QXDM default)
3. Search using the Alternate Script Path (Options->Settings)
Example RequestItemFile "Call Origination Request" "CallOriginate.txt"
Builds and sends a "Call Origination Request" to the target with field values contained in the file "CallOriginate.txt" and waits for and parses the response using the default database description.

RequestItemFile "Mode Change Request" "ModeOnlineDigital.txt"
Builds and sends a "Mode Change Request" with field values contained in ModeOnlineDigital.txt to the target and waits for and parses the response using the default database description.
RequestNamedItem RequestNamedItem allows one to schedule a request to be sent to the phone. The request itself is built according to a description entered into the QXDM database or a user database. RequestNamedItem waits for a response and if received, parses it according to the description matching the provided DIAG entity name.
Syntax RequestNamedItem "ResponseName" "RequestName" Fields
"ResponseName" is the name of the DIAG entity used to parse the responses (if received)
"RequestName" is the name of the DIAG entity to request (entity must be either a DIAG request or subsystem dispatch request)
Fields is a list of space-separated field values used to build the request, if the request does not require field values this argument can be omitted
Example RequestNamedItem "Version Number Response" "Version Number Request"
Builds and sends a "Version Number Request" to the target and waits for and parses the response using the "Version Number Response" DIAG entity

RequestNamedItem "Extensible Parameter Retrieval Response" "Extensible Parameter Retrieval Request" 1 1 100
Builds and sends a "Extensible Parameter Retrieval Request" with field values 1, 1, and 100 to the target and waits for and parses the response using the "Extensible Parameter Retrieval Response" DIAG entity

WaitForItem WaitForItem allows you to register interest in and wait for an item. The item name is the DIAG entity name in the QXDM database or the user database. Requests for asynchronous commands (logs, events, and debug messages) may result in a registration being sent to the phone. WaitForItem can also be used to wait for synchronous requests or responses. At the conclusion of the command, the registration is revoked

NOTE: WaitForItem only waits for a response. It does not make a request. To generate requests use one of the above commands
Syntax WaitForItem "ItemName"
"ItemName" is the name of the item to wait for
Example WaitForItem "EVENT_CALL_CONTROL_TERMINATED"
Note
Commands are provided to support waiting for, requesting, and parsing of any item that is described in the QXDM database. Additional support is provided by the 'Item Tester' application. The 'Item Tester' application allows one to explore the QXDM database (as well as all merged user databases) and to create functioning examples of the above commands. The 'Item Tester' is accessible from the QXDM Tools menu.

Example SPC "000000"
Sends the string "000000" to the target as a security code
ST Turns transmission of the signaling tone on or off
Syntax ST On | Off
Example ST On
Terminate Stops RCVS1 or WSTS (offline) analog tests and requests the counter values from the target
Syntax Terminate
Example Terminate
Timestamp Requests the system time value from the target
Syntax Timestamp
Example Timestamp
To_Data_Mode Requests the target switch from DIAG mode to data services (AT command processor) mode
Syntax To_Data_Mode
Example To_Data_Mode
TXMute Turns transmitter audio signal on or off
Syntax TXMute On | Off
Example TXMute On
Wait Halts scripting for a specified amount of time
Syntax Wait Seconds
Example Wait 10
Halt script processing for 10 seconds
WSTS Start the (offline) analog WSTS test
Syntax WSTS
Example WSTS
